# Варианты развертывания и сетевые режимы Traefik Proxy

Репозиторий поддерживает **4 архитектурных режима запуска** в зависимости от требований безопасности вашей инфраструктуры (Root vs
Rootless) и способа взаимодействия с демоном контейнеризации (UNIX-сокет vs TCP-порт).

По умолчанию символическая ссылка `docker-compose.yml` указывает на наиболее безопасный и изолированный режим
`docker-compose.tcp-rootless.yml`.

---

## 🛠 Матрица совместимости режимов

| Конфигурационный файл                    | Демон (Хост)    | Контейнер        | Способ связи                | Безопасность    | Особенности                                                           |
|:-----------------------------------------|:----------------|:-----------------|:----------------------------|:----------------|:----------------------------------------------------------------------|
| **`docker-compose.socket-root.yml`**     | Root Docker     | `root` (0:0)     | `/var/run/docker.sock`      | ⚠️ Низкая       | Классический режим. Удобен для тестов, опасен в продакшене.           |
| **`docker-compose.socket-rootless.yml`** | Rootless Docker | `traefik` (1000) | `.../user/1000/docker.sock` | 🔒 Высокая      | Проброс пользовательского сокета хоста без рутовых прав.              |
| **`docker-compose.tcp-root.yml`**        | Root Docker     | `root` (0:0)     | TCP-порт (`2375`)           | 🛡 Средняя      | Полная изоляция от файловой системы хоста. Требует TLS/Фаервол.       |
| **`docker-compose.tcp-rootless.yml`**    | Rootless/Root   | `traefik` (1000) | TCP-порт (`2375`)           | 🛑 Максимальная | **Hardened-режим**. Нет root-прав, нет прямого доступа к сокет-файлу. |

---

## 📑 Подробное описание режимов и их настройка

### 1. Режимы сокета (Socket-based)

Взаимодействие с Docker API происходит через стандартный файл UNIX-сокета. Это обеспечивает максимальную производительность, но
накладывает ограничения на права доступа.

#### А. Стандартный Root-режим (`docker-compose.socket-root.yml`)

Контейнер работает от имени `root`, а сокет мапится напрямую из хост-системы:

```yaml
volumes:
  - /var/run/docker.sock:/var/run/docker.sock:ro
```

> ⚠️ **ВНИМАНИЕ:** Если злоумышленник скомпрометирует контейнер Traefik, наличие root-сокета позволит ему выполнить «побег» на
> хост-систему с полными root-правами. Рекомендуется использовать только в изолированных средах разработки.

#### Б. Пользовательский Rootless-режим (`docker-compose.socket-rootless.yml`)

Если на хосте настроен Docker в Rootless-режиме, стандартного файла `/var/run/docker.sock` в системе нет. Демон создает сокет в
пространстве пользователя (UID 1000).
Конфигурация пробрасывает именно пользовательский сокет:

```yaml
volumes:
  - /run/user/1000/docker.sock:/var/run/docker.sock:ro
```

* **Права доступа:** Контейнер запускается под пользователем `user: traefik`. Поскольку UID пользователя на хосте и внутри
  контейнера совпадает (1000), Traefik имеет легитимный доступ к сокету без необходимости использования директив типа `group_add`.

---

### 2. Сетевые режимы (TCP-based)

Позволяют полностью изолировать контейнер от файловой системы хоста. Связь с Docker API осуществляется по протоколу TCP через порт
`2375`. Это исключает возможность атаки через монтирование сокет-файлов.

#### 🚨 ВАЖНО: Предварительная настройка Systemd на хосте

По умолчанию Docker-демон слушает только локальный UNIX-сокет. Попытка привязать порт к `127.0.0.1` приведет к тому, что
контейнеры из виртуальной сети моста (например, `172.17.0.1`) не смогут достучаться до хоста, так как ядро Linux будет сбрасывать
пакеты.

Для корректной работы необходимо настроить Docker на прослушивание интерфейса `0.0.0.0`, обеспечив защиту фаерволом.

**Инструкция по настройке:**

1. Выполните команду переопределения юнита:
   ```bash
   sudo systemctl edit docker.service
   ```
2. Вставьте следующий блок конфигурации, сбрасывающий стандартные флаги запуска:
   ```ini
   [Service]
   ExecStart=
   ExecStart=/usr/bin/dockerd -H unix:///var/run/docker.sock -H tcp://0.0.0.0:2375 --containerd=/run/containerd/containerd.sock
   ```
3. Остановите и отключите встроенный сокет-активатор systemd, чтобы избежать конфликтов за файл сокета:
   ```bash
   sudo systemctl stop docker.socket
   ```
4. Примените настройки и перезапустите службу:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl restart docker
   ```

> 🔒 **БЕЗОПАСНОСТЬ (КРИТИЧЕСКИ ВАЖНО):** Так как порт `2375` теперь слушает интерфейс `0.0.0.0`, он открыт для всего мира. Если у
> вашего сервера есть публичный IP, вы **обязаны** закрыть этот порт для внешних подключений в фаерволе!
> * Для **UFW** (Ubuntu): `sudo ufw deny 2375/tcp` (UFW по умолчанию разрешает трафик внутри моста `docker0`, поэтому Traefik
    подключится, а внешние атаки будут заблокированы).
> * Для **nftables / firewalld**: убедитесь, что порт `2375` разрешен только для зоны `docker0`.

После этого в файлах `docker-compose.tcp-*.yml` магия `host-gateway` динамически пропишет нужный IP-адрес хоста в контейнер:

```yaml
environment:
  - "TRAEFIK_PROVIDERS_DOCKER_ENDPOINT=http://host.docker.internal:2375"
extra_hosts:
  - "host.docker.internal:host-gateway"
```

## 🦫 Особенности работы с Podman

Если вы используете **Podman** вместо Docker, архитектура взаимодействия с сокетами упрощается благодаря нативной поддержке
Rootless-режима, но имеет свои нюансы.

### Запуск службы сокета Podman

Podman не имеет постоянно запущенного демона. Чтобы предоставить сокет для Traefik, необходимо активировать сервис сокета для
конкретного пользователя:

```bash
# Запуск сокета в пространстве пользователя (Rootless)
systemctl --user enable --now podman.socket

# Проверка пути к сокету
echo $XDG_RUNTIME_DIR/podman/podman.sock
```

### Настройка `docker-compose.yml` под Podman:

1. Выберите режим `docker-compose.socket-rootless.yml`.
2. Замените путь монтирования сокета на стороне хоста на путь к сокету Podman:
   ```yaml
   volumes:
     - /run/user/1000/podman/podman.sock:/var/run/docker.sock:ro
   ```
3. **Флаг безопасности SELinux:** Если вы работаете на RHEL/Fedora/CentOS, добавьте флаг `:z` или `:Z` к монтируемым томам
   конфигурации и сокета, чтобы Podman автоматически скорректировал контексты безопасности SELinux.
4. Переменная окружения `TRAEFIK_PROVIDERS_DOCKER_ENDPOINT=/var/run/docker.sock` остается неизменной, так как Podman полностью
   эмулирует API Docker.

## 🚀 Быстрый старт

1. Выберите желаемый режим (например, TCP Rootless) и обновите символическую ссылку:
   ```bash
   ln -sf docker-compose.tcp-rootless.yml docker-compose.yml
   ```
2. Убедитесь, что внешняя публичная сеть создана:
   ```bash
   docker network create traefik_public
   ```
3. Запустите стек:
   ```bash
   docker compose up -d --build
   ```
